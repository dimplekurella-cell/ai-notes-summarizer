# Contributing to AI Notes Summarizer

Thank you for your interest in contributing to **AI Notes Summarizer**! We welcome contributions from developers, students, and open-source enthusiasts who would like to improve the project.

## How to Contribute

### 1. Fork the Repository

Create your own copy of the project by forking the repository.

### 2. Clone the Repository

Clone your fork to your local machine:

```bash
git clone https://code.swecha.org/jaruplaswetha/ai-notes-summarizer.git
cd ai-notes-summarizer
```

### 3. Create a New Branch

Create a separate branch for your feature or bug fix:

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes

* Follow clean and readable coding practices.
* Add comments where necessary.
* Ensure your changes do not break existing functionality.

### 5. Test Your Changes

Before submitting, make sure the application runs correctly:

```bash
streamlit run app.py
```

Verify that:

* File upload works correctly.
* Notes are processed successfully.
* Summaries and key points are generated as expected.

### 6. Commit Your Changes

Write clear and descriptive commit messages:

```bash
git add .
git commit -m "Add: brief description of the changes"
```

### 7. Push and Create a Merge Request

```bash
git push origin feature/your-feature-name
```

Then create a **Merge Request (MR)** for review.

---

## Contribution Guidelines

* Keep code simple, modular, and well-documented.
* Follow Python coding standards (PEP 8).
* Do not commit sensitive information or credentials.
* Update the documentation if your changes affect project usage.
* Be respectful and collaborative when discussing issues and reviewing code.

---

## Reporting Issues

If you find a bug or have a feature request, please create an issue with:

* A clear title and description.
* Steps to reproduce the problem (if applicable).
* Expected and actual behavior.
* Screenshots or logs, if available.

---

## Code of Conduct

By participating in this project, you agree to be respectful and constructive. Please maintain a welcoming and inclusive environment for all contributors.
